# 사용자 시나리오 초안

## 등장인물 (Actors)

| 역할             | 설명                                         |
| ---------------- | -------------------------------------------- |
| **크리에이터**   | 강의를 개설하고 관리하는 강사                |
| **클래스메이트** | 강의를 수강 신청하는 수강생                  |
| **시스템**       | 정원 관리, 상태 전이 등 비즈니스 규칙을 집행 |

---

## 시나리오 1: 강의 등록 및 상태 전이

### 1-1. 크리에이터가 강의를 초안으로 등록한다

**사전 조건:** 크리에이터가 로그인되어 있다.

**흐름:**

1. 크리에이터가 제목, 설명, 가격(원 단위 정수, 소수 없음), 정원, 수강 기간(시작 일시~종료 일시)을 입력하여 강의를 등록한다.
2. 시스템은 강의를 `DRAFT` 상태로 저장한다.
3. `DRAFT` 상태에서는 클래스메이트가 해당 강의에 수강 신청을 할 수 없다.

**예외:**

- 필수 항목(제목, 가격, 정원, 수강 기간 일시)이 누락된 경우 → 400 Bad Request

---

### 1-2. 크리에이터가 강의를 모집 중(OPEN)으로 전환한다

**사전 조건:** 강의가 `DRAFT` 상태이다.

**흐름:**

1. 크리에이터가 강의를 `OPEN` 상태로 변경한다.
2. 시스템은 상태를 `OPEN`으로 갱신한다.
3. 이후 클래스메이트가 해당 강의에 수강 신청을 할 수 있다.

---

### 1-3. 크리에이터가 강의를 모집 마감(CLOSED)으로 전환한다

**사전 조건:** 강의가 `OPEN` 상태이다.

**흐름:**

1. 크리에이터가 강의를 `CLOSED` 상태로 변경한다.
2. 시스템은 상태를 `CLOSED`로 갱신한다.
3. 이후 신규 수강 신청은 불가하다.

**예외:**

- `DRAFT` → `CLOSED` 와 같이 허용되지 않는 상태 전이를 시도하는 경우 → 400 Bad Request

---

## 시나리오 2: 강의 목록 및 상세 조회

### 2-1. 클래스메이트가 강의 목록을 조회한다

**흐름:**

1. 클래스메이트가 강의 목록을 요청한다.
2. 클래스메이트는 상태 필터(`OPEN` / `CLOSED`)를 선택적으로 적용할 수 있다.
3. 시스템은 조건에 맞는 강의 목록을 반환한다.

---

### 2-2. 클래스메이트가 강의 상세 정보를 조회한다

**흐름:**

1. 클래스메이트가 특정 강의의 상세 조회를 요청한다.
2. 시스템은 강의 정보(제목, 설명, 가격, 정원, 수강 기간 일시)와 함께 **현재 신청 인원**을 반환한다.
   - 현재 신청 인원 = `PENDING` + `CONFIRMED` 상태의 수강 신청 수

**예외:**

- 존재하지 않는 강의 ID → 404 Not Found

---

## 시나리오 3: 수강 신청

### 3-1. 클래스메이트가 정상적으로 수강 신청을 한다

**사전 조건:**

- 강의가 `OPEN` 상태이다.
- 현재 신청 인원이 정원 미만이다.
- 해당 클래스메이트가 동일 강의에 이미 신청하지 않은 상태이다.

**흐름:**

1. 클래스메이트가 강의에 수강 신청을 요청한다.
2. 시스템은 신청을 `PENDING` 상태로 저장한다.
3. 클래스메이트는 결제를 진행해야 수강이 확정된다.

**예외:**

- 강의가 `DRAFT` 또는 `CLOSED` 상태인 경우 → 신청 불가 (400 / 409)
- 이미 동일 강의에 신청한 경우 → 중복 신청 불가 (409 Conflict)
- 정원이 가득 찬 경우 → 시나리오 4 참고

---

### 3-2. 클래스메이트가 결제를 완료하여 수강을 확정한다

**사전 조건:** 수강 신청이 `PENDING` 상태이다.

**흐름:**

1. 클래스메이트가 결제 확정을 요청한다. (외부 결제 시스템 연동 없이 단순 상태 변경)
2. 시스템은 수강 신청 상태를 `CONFIRMED`로 변경한다.
3. 취소 가능 기간(결제 후 7일)이 시작된다.

**예외:**

- 이미 `CONFIRMED` 또는 `CANCELLED` 상태인 경우 → 상태 변경 불가 (400)

---

## 시나리오 4: 정원 초과 방지

### 4-1. 정원이 꽉 찬 강의에 수강 신청을 시도한다

**사전 조건:** 강의의 현재 신청 인원 = 정원.

**흐름:**

1. 클래스메이트가 해당 강의에 수강 신청을 요청한다.
2. 시스템은 현재 신청 인원이 정원 이상임을 확인하고 신청을 거부한다.

**예외:** → 409 Conflict (정원 초과)

---

### 4-2. 동시에 여러 클래스메이트가 마지막 자리에 신청한다

**사전 조건:** 강의의 잔여 정원이 1석이다.

**흐름:**

1. 클래스메이트 A와 B가 동시에 해당 강의에 수강 신청을 요청한다.
2. 시스템은 동시성 제어(비관적 락 또는 낙관적 락)를 통해 단 1명만 신청을 성공시킨다.
3. 나머지 1명은 정원 초과로 거부된다.

**핵심 보장:** 동시 요청이 들어와도 정원을 초과한 신청은 절대 허용되지 않는다.

---

## 시나리오 5: 수강 취소

### 5-1. 클래스메이트가 취소 가능 기간 내에 수강을 취소한다

**사전 조건:**

- 수강 신청이 `CONFIRMED` 상태이다.
- 결제 확정 후 7일 이내이다.

**흐름:**

1. 클래스메이트가 수강 취소를 요청한다.
2. 시스템은 수강 신청 상태를 `CANCELLED`로 변경한다.
3. 해당 강의의 현재 신청 인원이 1 감소한다.
4. 대기열이 있다면 대기열의 첫 번째 클래스메이트에게 신청 기회가 부여된다. (시나리오 8 참고)

---

### 5-2. 클래스메이트가 취소 가능 기간이 지난 후 수강을 취소하려 한다

**사전 조건:**

- 수강 신청이 `CONFIRMED` 상태이다.
- 결제 확정 후 7일이 초과되었다.

**흐름:**

1. 클래스메이트가 수강 취소를 요청한다.
2. 시스템은 취소 가능 기간이 지났음을 확인하고 취소를 거부한다.

**예외:** → 400 Bad Request (취소 가능 기간 초과)

---

### 5-3. 아직 결제하지 않은(PENDING) 신청을 취소한다

**사전 조건:** 수강 신청이 `PENDING` 상태이다.

**흐름:**

1. 클래스메이트가 수강 취소를 요청한다.
2. 시스템은 수강 신청 상태를 `CANCELLED`로 변경한다.
   - `PENDING` → `CANCELLED`는 7일 제한 없이 가능하다.

---

## 시나리오 6: 내 수강 신청 목록 조회

### 6-1. 클래스메이트가 자신의 수강 신청 목록을 조회한다

**흐름:**

1. 클래스메이트가 본인의 수강 신청 목록을 요청한다.
2. 시스템은 해당 사용자의 모든 수강 신청 내역(강의 정보 + 신청 상태)을 반환한다.
3. 목록은 페이지네이션으로 제공된다. (선택 구현)

---

## 시나리오 7: 강의별 수강생 목록 조회 (크리에이터 전용, 선택 구현)

### 7-1. 크리에이터가 자신의 강의 수강생 목록을 조회한다

**사전 조건:** 조회하려는 강의의 크리에이터 본인이다.

**흐름:**

1. 크리에이터가 특정 강의의 수강생 목록을 요청한다.
2. 시스템은 해당 강의의 `CONFIRMED` 상태 수강생 목록을 반환한다.

**예외:**

- 존재하지 않는 강의 ID → 404 Not Found

---

## 시나리오 8: 대기열 (선택 구현)

### 8-1. 정원이 꽉 찬 강의에 대기 신청을 한다

**사전 조건:** 강의의 현재 신청 인원 = 정원.

**흐름:**

1. 클래스메이트가 해당 강의에 수강 신청을 요청한다.
2. 시스템은 정원 초과임을 감지하고 대기열에 등록한다.
3. 이후 누군가 취소하면 대기열의 첫 번째 클래스메이트에게 신청 기회가 부여된다.

---

## 상태 전이 요약

### 강의 상태

```
DRAFT → OPEN → CLOSED
```

### 수강 신청 상태

```
PENDING → CONFIRMED → CANCELLED
PENDING → CANCELLED
```

---

## 엣지 케이스 체크리스트

- [ ] `PENDING` 상태에서 결제 없이 취소하는 경우
- [ ] 동일 강의 중복 신청 시도
- [ ] `DRAFT` / `CLOSED` 강의에 신청 시도
- [ ] 정원 1석에 동시 다수 신청 (동시성)
- [ ] 결제 후 정확히 7일째 되는 날 취소 시도 (경계값)
- [ ] 본인 강의가 아닌 강의 수강생 목록 조회 시도
- [ ] 존재하지 않는 강의 ID로 요청
